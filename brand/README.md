# brand/ — the canonical brand source

Every brand fact lives here, once. One fact per file. Each file opens with light YAML frontmatter
(machine layer) above prose (human layer).

## Core (text)

| File | What it defines |
| --- | --- |
| `foundation.md` | Mission, vision, market gap, positioning statement, the four-phase system. |
| `beliefs.md` | What the brand is convinced of. |
| `values.md` | Core operating principles. |
| `audience.md` | Who it's for. |
| `voice.md` | Voice & tone: principles, do/don't, register per surface. |
| `lexicon.md` | Words we use / ban; naming conventions for asset labels. |
| `messaging.md` | Pillars, key messages per product/phase, approved taglines. |
| `anti-patterns.md` | What the brand avoids — in words and visuals. |

## Visual (`visual/`)

| File | What it defines |
| --- | --- |
| `identity.md` | The tube system, label system, phase ↔ product map, formula/texture per product. |
| `color.md` | Phase hex codes, photography background hex codes. |
| `typography.md` | The three-typeface system and when to use each. |
| `treatment.md` | Shared lighting/focus/tone rules across all imagery. |
| `photography-product.md` | Composition rules for texture/formula refs vs. finished product shots. |
| `photography-people.md` | Who, crop/framing, expression, product-on-skin, skin rendering rules. |
